# labforge SIEM — static live demo

A **VM-free, in-browser** demo of the labforge SIEM detection lab. Recruiters
and reviewers click one link and see the real dashboard — host grid, per-host
drilldown with highlighted attack evidence, cross-host search, a severity
chart, and the automatic **attack detections** — without spinning up 8 GB of
virtual machines.

It is a single self-contained page (`index.html`) plus a generated data file
(`data.js`). No build step, no framework, no external requests — drop it on any
static host (GitHub Pages, Netlify, a plain web server) and it works offline.

## What it shows

The embedded corpus is a canned but realistic attack story:

1. an **nmap** port sweep across the segment,
2. a **hydra** SSH brute force against DVWA (`Failed password` storm),
3. an **sqlmap** injection run against DVWA,
4. a **nikto** web scan against Juice Shop, and
5. an **RDP/SMB brute force** against the Windows victim (an `EventID=4625`
   storm) that finally lands a successful `4624` — flagged **critical**.

## Honesty: it can't drift from the real product

`data.js` is generated by [`build_demo.py`](build_demo.py), which imports the
**real** `labforge_siem` package and runs the **actual** detection engine over
the **same** deterministic corpus the flagship tests assert on. So the findings
you see in the browser are byte-for-byte what the live SIEM box would compute.

## Rebuild

```bash
python demo/build_demo.py     # regenerates demo/data.js
# then open demo/index.html (or serve the folder):
python -m http.server 8000 --directory demo
```

CI regenerates `data.js` and fails if the committed copy is stale, so the demo
stays in lock-step with the engine.
